Десктопное приложение для Windows, которое автоматически следит за выбранной Wi-Fi сетью, удерживает подключение к ней и контролирует доступ в интернет.

Живёт в системном трее и не мешает работе — пока соединение в порядке, о нём можно забыть.

Программа особенно полезна в местах с нестабильным Wi-Fi сигналом (например, за городом
или в зонах слабого покрытия) — она самостоятельно переподключается при разрыве связи.

> Это **Flutter-порт** [wifi-monitor-python](../wifi-monitor-python) на Dart/Flutter Desktop. Поведение, интерфейс, цвета, размеры окна и тексты сообщений сохранены 1:1 с Python-версией. В будущем планируется добавить сборку под Android.

## Как пользоваться

При первом запуске введите SSID (имя сети) и пароль.
Далее используйте кнопки в интерфейсе для запуска/остановки мониторинга и очистки лога.
Закрытие окна сворачивает приложение в трей — для полного выхода используйте меню трея.

## Назначение

Wi-Fi Монитор решает задачу **стабильного и автоматического подключения** к конкретной Wi-Fi сети на Windows-устройстве.

**Проблема:** в местах со слабым покрытием (загородный дом, дача, удалённый офис, помещения с толстыми стенами) Wi-Fi регулярно «отваливается». Windows далеко не всегда переподключается сама, особенно если сеть на время полностью пропадает из эфира — пользователю приходится вручную кликать по значку сети, выбирать сеть и ждать.

**Решение:** компактная утилита, которая постоянно сканирует эфир на наличие нужного SSID, переподключается при разрыве, проверяет реальный доступ в интернет и показывает живой статус задержки до 8.8.8.8 — всё это из системного трея, без лишних окон и кликов.


## Функциональность

### Мониторинг и автоматическое переподключение
- Сканирование эфира через `netsh wlan show networks` с заданным интервалом.
- Автоматическое создание профиля сети (WPA2-PSK / AES) и подключение через `netsh wlan connect`.
- До **100 попыток переподключения** с активным ожиданием результата, без фиксированных пауз.
- Корректная работа в локалях Windows на русском и английском (распознавание маркеров «Подключено» / «Connected»).

### Проверка интернета и пинг-монитор
- Двухуровневая проверка интернета: сначала пробуем роутер, при неудаче — DNS Google (8.8.8.8).
- Отдельный изолят (`Isolate`) пинга 8.8.8.8 с настраиваемым интервалом и цветовой индикацией качества связи:
    - 🟢 **< 80 мс** — отлично
    - 🟡 **< 200 мс** — нормально
    - 🔴 **> 200 мс** — плохо
- Автодетект активного VPN (время отклика < 1 мс) с отображением «VPN is ON».

### Системный трей
- При закрытии окна приложение **сворачивается в трей**, а не выгружается из памяти.
- Цветная иконка отражает текущее состояние подключения: 🟢 зелёная — подключено, 🔴 красная — нет соединения.
- Контекстное меню трея: «Открыть», «Выход». Правый клик по иконке открывает меню.
- Всплывающая подсказка показывает текущий SSID и его статус.

### Безопасное хранение учётных данных
- SSID и флаг «Запомнить меня» хранятся в JSON-файле `%APPDATA%\WiFiMonitor\settings.json`.
- Пароль сохраняется в **Windows Credential Manager** через `flutter_secure_storage` (DPAPI) — в открытом виде он нигде не лежит.
- Полное удаление сохранённых данных одним кликом (снять галку «Запомнить меня» при следующем запуске).

### Настройки пользователя
- Интервал проверки Wi-Fi (1–3600 сек).
- Интервал пинга (1–3600 сек).
- IP роутера для проверки локальной сети (значение по умолчанию: `192.168.0.1`).
- Изменения применяются на лету — сервисы мониторинга и пинга пересоздаются без необходимости перезагружать приложение.

### Лог событий
- Журнал событий в главном окне с временными метками.
- **Дедупликация повторяющихся строк:** при повторе статуса обновляется только время в существующей строке.
- Автоматическое обрезание лога до **500 последних строк** для экономии памяти.
- Кнопка очистки в один клик.

## Технологический стек

| Категория | Технология |
|-----------|-----------|
| **Язык разработки** | Dart 3.12+ |
| **GUI-фреймворк** | Flutter 3.44+ (Material, Fusion-подобная светлая тема) |
| **Управление сетью** | Windows `netsh wlan` (через `Process.run`) |
| **Хранение пароля** | `flutter_secure_storage` (Windows Credential Manager / DPAPI) |
| **Хранение настроек** | JSON-файл в `%APPDATA%\WiFiMonitor\` |

| **Системный трей** | `tray_manager` |
| **Управление окном** | `window_manager` (фикс. размер, прячем в трей) |
| **Многопоточность** | `Stream` + `Isolate` (пинг), отменяемые задержки |
| **Сборка** | `flutter build windows` (MSVC 2022, C++ workload) |
| **Платформа** | Windows 10 / 11 (в перспективе — Android) |

## Сборка, запуск и дистрибуция

### Требования к окружению
- Windows 10 / 11
- [Flutter SDK 3.44+](https://docs.flutter.dev/get-started/install/windows) (`flutter --version`)
- **Visual Studio 2022** с workload **«Desktop development with C++»** (нужен MSVC + Windows SDK + CMake — иначе `flutter build windows` не запустится).
- Wi-Fi адаптер, поддерживаемый Windows (используется системный `netsh wlan`).

Проверить готовность окружения:
```bash
flutter doctor
flutter config --enable-windows-desktop
```

### Быстрый запуск из исходников

1. **Клонировать репозиторий:**
   ```bash
   git clone https://github.com/<your-account>/wifi-monitor-flutter.git
   cd wifi-monitor-flutter
   ```

2. **Установить зависимости:**
   ```bash
   flutter pub get
   ```

3. **Запустить:**
   ```bash
   flutter run -d windows
   ```

### Сборка EXE-файла

```bash
flutter build windows --release
```

Готовый исполняемый файл и все необходимые DLL/ассеты будут лежать в:
```
build\windows\x64\runner\Release\
├── wifi_monitor.exe
├── flutter_windows.dll
├── *.dll (плагины: tray_manager, window_manager, secure_storage, win32_registry)
└── data\flutter_assets\assets\tray_green.png, tray_red.png
```

Распространять нужно **всю** папку `Release\` — `wifi_monitor.exe` без соседних DLL и `data/` не запустится. Для одного файла используйте упаковщик (например, [MSIX](https://pub.dev/packages/msix) или Inno Setup).

### Открыть в Android Studio / IntelliJ

1. **File → Open** → выбрать папку `wifi-monitor-flutter`.
2. IDE предложит «Get dependencies» — согласиться (или вручную `flutter pub get`).
3. В выпадающем списке устройств выбрать **Windows (desktop)** и нажать **Run**.
   - Если устройство не появляется: `flutter config --enable-windows-desktop` и перезапустить IDE.

## Как пользоваться

1. **Первый запуск.** Введите SSID нужной сети и пароль. Поставьте галку «Запомнить меня», чтобы при следующем запуске данные подставились автоматически (пароль шифруется в Credential Manager через DPAPI).
2. **Главное окно.** Мониторинг стартует сразу. В верхней части — текущий пинг до 8.8.8.8 с цветовой индикацией, в центре — журнал событий, внизу — строка состояния и кнопки управления.
3. **Кнопки управления.**
    - **Запуск мониторинга** / **Остановить** — управление фоновым сервисом проверки сети.
    - **Очистить лог** — сброс журнала.
    - **Настройки** — изменение интервалов проверки и пинга на лету.
4. **Свернуть в трей.** Закрытие окна (крестик) скрывает приложение в системный трей — мониторинг продолжает работать. Для полного выхода используйте пункт «Выход» в меню трея.
5. **Автозапуск.** Если запустить `.exe` с аргументом `--autostart` (так делает запись в `HKCU\...\Run`), главное окно стартует скрытым — будет видна только иконка в трее.

## Что нового в v0.3.5 (Flutter-порт)

- 🆕 **Полный перенос на Flutter/Dart** — единая кодовая база, готовая к расширению на Android.
- 🆕 **Нативный Flutter Desktop под Windows** — один `.exe` + папка ассетов, без интерпретатора Python.
- 🔄 **JSON-конфиг вместо реестра** для пользовательских настроек (`%APPDATA%\WiFiMonitor\settings.json`) — переносимо и читаемо вручную.
- 🔒 **Пароль в Credential Manager через `flutter_secure_storage`** (тот же DPAPI, что у `keyring` в Python-версии).
- 🧵 **Стримы + Isolate** вместо `QThread` — отмена мониторинга и пинга мгновенная, без фиксированных пауз.
- 🪟 **`window_manager` + `tray_manager`** заменили `QSystemTrayIcon`/`QMainWindow`, поведение «закрытие → в трей» сохранено 1:1.

### Сохранено из v0.3.5 Python-версии
- Точно такой же интерфейс: размеры (600×400), палитра, шрифты (Arial / Consolas), тексты и расположение кнопок.
- Те же интервалы по умолчанию и те же диапазоны в настройках (1–3600 сек).
- Дедупликация лога и лимит 500 строк.
- Двухуровневая проверка интернета (роутер → 8.8.8.8) и автодетект VPN.
- Поддержка русской и английской локали Windows.
